# Scope

## Goal

현재 단계에서 무엇을 계속 유지하고, 무엇을 다음 작업에서 다루지 않을지 명확히 정의한다.

## In Scope

- Electron 기반 Windows, macOS 데스크톱 클라이언트
- 사내 인증 서버 프록시 로그인
- JWT Access Token, Refresh Token 기반 인증 흐름
- Redis 기반 세션 관리와 중복 로그인 제어
- STOMP 기반 실시간 채팅
- 1:1 DM과 그룹 채팅
- 메시지 조회, 읽음 처리, 안읽은 메시지 수 반영
- 타이핑 인디케이터
- 파일 업로드와 다운로드
- 공지 브로드캐스트와 특정 사용자 대상 내부 알림
- Electron 네이티브 알림과 시스템 트레이 배지
- 사용자 목록 조회와 DM 진입

## Out Of Scope

- 모바일 앱 개발
- 공개 회원가입과 외부 사용자 초대
- 메신저 내부 비밀번호 저장
- 음성 통화, 영상 통화, 화면 공유
- 메시지 검색 고도화
- 다국어 지원
- 관리자 전용 백오피스
- 새로운 상태관리 라이브러리나 UI 프레임워크 도입
- 프론트엔드가 직접 사내 인증 서버를 호출하는 구조

## Current Phase

- 구현 완료 기준으로는 초기 핵심 기능 묶음이 이미 반영된 상태다.
- 현재 우선순위는 신규 기술 도입보다 기능 안정화, 문서 정리, 테스트 보강, UX 개선에 있다.

## Priority Rules

1. 안정성과 보안
2. 현재 아키텍처 일관성 유지
3. 사용자 경험 개선
4. 코드 미려함과 추상화

## Non-Negotiables

- `contextIsolation: true`와 `nodeIntegration: false`를 유지한다.
- 사내 인증 호출은 반드시 백엔드에서만 수행한다.
- 내부 API 키와 비밀값은 코드에 하드코딩하지 않는다.
- 기존 Electron + React + Spring Boot 구조를 무너뜨리는 큰 리라이트는 하지 않는다.
