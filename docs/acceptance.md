# Acceptance

## Goal

작업이 끝났는지 판단하는 기준을 기능별로 정리한다.

## Global Done Criteria

- 변경 범위가 문서화된 요구사항과 일치한다.
- 기존 아키텍처 규칙과 보안 규칙을 깨지 않는다.
- 타입, 빌드, 테스트 또는 해당 범위의 검증 결과가 설명 가능하다.
- 실패 케이스에 대한 동작이 명확하다.

## Authentication

- 올바른 사내 계정으로 로그인할 수 있다.
- 잘못된 자격 증명은 적절한 에러로 처리된다.
- Refresh Token 기반 재인증이 동작한다.
- 중복 로그인 시 기존 세션이 만료되고 사용자에게 반영된다.

## Chat

- 채널 목록과 메시지 목록이 조회된다.
- 메시지 전송 후 본인과 상대방 화면에 반영된다.
- 타이핑 상태가 필요한 범위에서 표시된다.
- 읽음 처리와 안읽은 메시지 수가 일관되게 갱신된다.

## File

- 허용된 파일 형식과 크기 내에서 업로드 가능하다.
- 인증 사용자만 다운로드 가능하다.
- 이미지 첨부는 가능한 경우 미리보기 된다.

## Notifications And Presence

- 사용자 상태 조회가 가능하다.
- 네이티브 알림과 앱 내 알림 목록이 동작한다.
- 미읽음 배지와 트레이 상태가 주요 이벤트에 따라 갱신된다.

## Internal Integrations

- 내부 API 키가 없거나 잘못되면 요청이 거부된다.
- 전체 공지는 저장과 브로드캐스트가 함께 처리된다.
- 특정 사용자 알림은 온라인과 오프라인 케이스 모두 처리된다.

## Documentation

- 새로 추가된 기능이나 규칙은 관련 `docs/` 파일에 반영된다.
- 마스터 프롬프트는 장기 명세를 중복 작성하지 않고 `docs/`를 참조한다.
