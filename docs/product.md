# Product

## Goal

사내 사용자를 위한 실시간 메신저 데스크톱 애플리케이션을 제공한다. 사용자는 Windows와 macOS 환경에서 안전하게 로그인하고, 1:1 대화와 그룹 채팅, 파일 공유, 공지와 외부 알림을 하나의 앱에서 처리할 수 있어야 한다.

## Problem

- 사내 커뮤니케이션과 시스템 알림이 여러 채널로 분산되어 있다.
- 외부 사내 시스템의 이벤트를 실시간으로 사용자에게 전달하기 어렵다.
- 사내 인증 체계와 연동되면서도 메신저 자체 세션은 안전하게 관리해야 한다.

## Target Users

- 사내 직원
- 공지나 업무 알림을 발송하는 사내 시스템 운영자
- 메신저 백엔드와 연동하는 내부 서비스 담당자

## Product Outcomes

- 사용자는 사내 계정으로 로그인해 바로 채팅을 시작할 수 있다.
- 메시지, 파일, 알림 이력이 앱 내에서 일관되게 조회된다.
- 외부 시스템 공지와 사용자 대상 알림이 메신저 안으로 통합된다.
- 중복 로그인, 세션 만료, 오프라인 복구 같은 운영 이슈에 안정적으로 대응한다.

## Core User Flows

### 1. 로그인과 자동 재인증

- 사용자는 사내 계정으로 로그인한다.
- 백엔드는 사내 인증 서버를 통해 자격을 검증한다.
- 성공 시 메신저 전용 Access Token과 Refresh Token을 발급한다.
- 앱 재시작 후에도 Refresh Token으로 자동 로그인할 수 있어야 한다.

### 2. 채널 기반 커뮤니케이션

- 사용자는 사용자 목록에서 다른 사용자를 선택해 1:1 DM으로 진입한다.
- 사용자는 그룹 채팅방을 만들고 멤버를 초대하거나 퇴장시킬 수 있다.
- 메시지 이력, 읽음 상태, 타이핑 상태가 실시간으로 반영되어야 한다.

### 3. 파일과 미디어 공유

- 사용자는 채팅창에서 파일을 첨부하거나 드래그앤드롭으로 업로드한다.
- 이미지 파일은 가능하면 인라인으로 미리보기 된다.
- 다운로드는 인증된 사용자만 가능해야 한다.

### 4. 알림과 Presence

- 사용자는 앱 내에서 공지, 개인 알림, 안읽은 메시지를 확인한다.
- 사용자의 온라인 상태는 다른 사용자에게 반영된다.
- 앱이 백그라운드일 때는 OS 네이티브 알림으로 중요한 이벤트를 볼 수 있어야 한다.

## Success Criteria

- 인증, 채팅, 파일, 알림, Presence 흐름이 로컬 개발 환경에서 끊김 없이 동작한다.
- 로그인 세션과 WebSocket 연결이 중복 로그인 정책을 일관되게 따른다.
- 외부 시스템에서 호출한 내부 API가 실시간 공지 또는 개인 알림으로 연결된다.
- 프론트엔드와 백엔드가 각자 독립적으로 빌드 및 검증 가능하다.

## Current Product State

- 프로젝트는 Electron + React 프론트엔드와 Spring Boot 백엔드로 구성되어 있다.
- 인증, 채팅, 파일 업로드, Presence, 공지/알림, 패키징의 기본 뼈대가 이미 구현되어 있다.
- 현재 문서 목적은 새 기능 추가보다 구현 기준을 분리해 AI 작업 품질을 안정화하는 데 있다.
